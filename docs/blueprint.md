# **App Name**: Decision Crafter

## Core Features:

- AHP Fundamentals Page: A dedicated section providing comprehensive explanations, definitions, and visual diagrams of the Analytic Hierarchy Process (AHP) method, ideal for student learning.
- Hierarchical Model Builder: Interactive tool allowing users to define decision criteria and alternatives, constructing the AHP hierarchy as shown in the example diagram.
- Pairwise Comparison Input: User-friendly interface for inputting pairwise comparisons between criteria and alternatives using a preference scale.
- AHP Calculation Engine: Backend logic to perform all necessary AHP calculations, including deriving weights from comparison matrices, synthesizing priorities, and checking consistency ratios.
- Interactive Results & Visualizations: Display of final AHP priority vectors, consistency ratio checks, and dynamic charts (e.g., bar charts for weights, network graphs for hierarchies) for better comprehension.
- Pre-built AHP Templates: A selection of pre-configured AHP decision models (e.g., 'Cloud Provider Selection') for students to explore, manipulate, and learn from existing examples.

## Style Guidelines:

- Primary color: A muted, deep plum (#725181) for an intellectual yet approachable foundation in a light scheme.
- Background color: A very subtle lavender (#F5F3F8), providing a clean, airy canvas for detailed content and clear readability.
- Accent color: A vibrant, clear blue (#4C4CE5), utilized for interactive elements, highlights, and calls to action, offering purposeful contrast.
- Headline font: 'Alegreya' (serif) for an elegant, intellectual feel; Body font: 'Inter' (sans-serif) for modern clarity and excellent readability in instructional content.
- Clean, geometrically informed icons with a slightly abstracted style, representing hierarchy, comparison, and decision nodes, reinforcing the analytical nature of the app.
- A well-structured, modular layout with generous whitespace to ensure readability for educational content and clear separation of interactive components. Emphasize a responsive grid system for flexibility across devices.
- Subtle, deliberate animations for component interactions and transitions, enhancing user feedback without distracting from the learning process, especially for visualizing AHP calculations and results.
